Portfolio Website
Amazon S3 and Route 53

The goal of this project was to host this very portfolio website using Amazon's aforementioned services.

Following are the steps which were followed to host the website with EC2 and Route 53:



  • Step 1: Login to AWS Console and access Route 53.
  • Step 2: Purchase a Domain Name by going to Registered Domains and searching for a domain name.
  • Step 3: Create an Amazon S3 bucket to host website files and label it as the domain name just acquired.
  • Step 4: Select the closest AWS region to reduce latency.
  • Step 5: Uncheck block public access (to allow others to view the website).
  • Step 6: Once the bucket has been created, upload all relevant files to the bucket.
  • Step 7: Enable static website hosting for the bucket properties.
  • Step 8: Add a bucket policy.
  • Step 9: Direct the domain to S3 bucket from Route 53.
  • Step 10: Issue a new security certificate through AWS Certificate Manager and enable it using either DNS or Email validation.
  • Step 11: Install the certifcate on cloud front and point S3 bucket to it.




This was a quick and simple project that allowed me to get cracking with some hands-on AWS experience.

Looking forward to exploring other AWS services to build projects and documenting my AWS journery further.



CI/CD Pipeline Setup
Amazon Codepipeline and S3

Once the website was hosted, it was vital to setup a CI/CD pipeline to ensure regular updates of the website with other projects I had in mind.

Following are the steps which were followed to setup the pipeline to connect my Github repo with the S3 bucket hosting site files:





  • Step 1: Login to AWS Console and access CodePipeline.
  • Step 2: Click Create pipline and assign a name to the pipeline.
  • Step 3: Select Github v1 as source provider in the Add source stage and connect to your repository by providing necessary permissions.
  • Step 4: Select the relevant code respository and branch and leave detection options set to Github webhooks.
  • Step 5: Since I will not be using any build tools, I skipped the Build Stage.
  • Step 6: Set the deploy provider to Amazon S3, Region to the zone matching our S3 bucket for website objects and select the website bucket.
  • Step 7: Check the "Extract file before deploy" option and click Next.
  • Step 8: After reviewing, create the pipeline.




Setting up a CI/CD pipeline is a crucial exercise in the world of DevOps and something that will pay dividends in the context of this project.

One thing to note here that I haven't mentioned explicitly thus far is that I am also using AWS cloudwatch for billing management and alarms.



Data Visualization
Amazon QuickSight and S3

I wanted to take a large dataset and use Amazon QuickSight for visualization.

Following are the steps which were followed to accomplish this goal:




  • Step 1: Login to AWS Console and access S3 service.
  • Step 2: Create a bucket by assigning it a name and choosing default settings.
  • Step 3: Upload the .csv file containing data (file obtained from BrightData) and manifest.json file for configuration.
  • Step 4: Once the bucket has been created and the data has been uploaded, access QuickSight service.
  • Step 5: Signed up for the QuickSight service and configured the account.
  • Step 6: At the bottom of the signup page, select the S3 bucket just created to be used with QuickSight.
  • Step 7: Once the QuickSight account is created, go to Datasets and import our dataset into QuickSight by clicking New Dataset.
  • Step 8: Select S3 from the options presented. Provide the URL for the manifest file from the S3 bucket.
  • Step 9: Go back to the QuickSight S3 data source window and paste the URL for the manifest file and give it a Data source name and click connect.
  • Step 10: Click visualize on the next window. In the New Sheet window, select the Interactive Sheet option and click create.
  • Step 11: Using the tools available in QuickSight, you can now play around with the data set and check out different visualizations.




This was a fun project that allowed me to get utilize AWS QuickSight to get down to playing with large data set and associated visualizations really quick.
