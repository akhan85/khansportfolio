Portfolio Website
Amazon EC2 and Route 53

The goal of this project was to host this very portfolio website using Amazon's aforementioned services.

Following are the steps which were followed to host the website with EC2 and Route 53:





  • Step 1: Login to AWS Console and access Route 53.
  • Step 2: Purchase a Domain Name by going to Registered Domains and searching for a domain name.
  • Step 3: Create an Amazon S3 bucket to host website files and label it as the domain name just acquired.
  • Step 4: Select the closest AWS region to reduce latency.
  • Step 5: Uncheck block public access (to allow others to view the website).
  • Step 6: Once the bucket has been created, upload all relevant files to the bucket.
  • Step 7: Enable static website hosting for the bucket properties.
  • Step 8: Add a bucket policy.
  • Step 9: Direct the domain to S3 bucket from Route 53.
  • Step 10: Issue a new security certificate through AWS Certificate Manager and enable it using either DNS or Email validation.
  • Step 11: Install the certifcate on cloud front and point S3 bucket to it.




This was a quick and simple project that allowed me to get cracking with some hands-on AWS experience.

Looking forward to exploring other AWS services to build projects and documenting my AWS journery further.
